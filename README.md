# rustFrida

ARM64 Android dynamic instrumentation framework.

## Project State

The currently verified mainline in this repository is:

- `spawn` and `attach`
- QuickJS scripting
- Java hook path backed by LSPlant
- device-side validation through `scripts/spawn_test.js`

The LSPlant core used by the Java hook path is vendored in-tree:

- `quickjs-hook/native/lsplant_core`

So a fresh machine does not need a separate LSPlant checkout.

## Requirements

- Rust toolchain
- Rust target: `aarch64-linux-android`
- Python 3
- Android SDK
- Android NDK
- Android SDK CMake package
- Android SDK Ninja package
- `adb`

Recommended environment variables:

- `ANDROID_HOME` or `ANDROID_SDK_ROOT`
- `ANDROID_NDK_HOME` or `ANDROID_NDK_ROOT`

On Windows, if `LIBCLANG_PATH` is not set, `build.py` will try to auto-detect it from the NDK.

## First-Time Checkout

QuickJS is kept as a git submodule, so a fresh clone still needs:

```bash
git submodule update --init --recursive quickjs-hook/quickjs-src
```

LSPlant is already vendored in-tree and does not require a separate checkout step.

## Quick Build

Use the repository-level build entry:

```bash
python build.py
```

This will:

1. Generate `.cargo/config.toml`
2. Build loader shellcode
3. Build `agent`
4. Build `rustfrida`

Expected outputs:

```text
target/aarch64-linux-android/release/libagent.so
target/aarch64-linux-android/release/rustfrida
```

## Manual Build

### 1. Generate Cargo cross-build config

```bash
python scripts/setup_cargo_config.py --force
```

### 2. Build loader shellcode

```bash
python loader/build_helpers.py
```

Expected outputs:

```text
loader/build/bootstrapper.bin
loader/build/rustfrida-loader.bin
```

### 3. Build QuickJS hook core

```bash
cargo build -p quickjs-hook --release --target aarch64-linux-android
```

### 4. Build agent

```bash
cargo build -p agent --release --target aarch64-linux-android
```

### 5. Build rustfrida

```bash
cargo build -p rust_frida --release --target aarch64-linux-android
```

## Deploy

```bash
adb push target/aarch64-linux-android/release/rustfrida /data/local/tmp/rustfrida
adb shell su -c "chmod 755 /data/local/tmp/rustfrida"
```

## Run

### Attach

```bash
/data/local/tmp/rustfrida --pid <pid>
/data/local/tmp/rustfrida --pid <pid> -l /data/local/tmp/script.js
```

### Spawn

```bash
/data/local/tmp/rustfrida --spawn com.example.app
/data/local/tmp/rustfrida --spawn com.example.app -l /data/local/tmp/script.js
```

### Inject by process name

```bash
/data/local/tmp/rustfrida --name com.example.app
```

### Verbose logging

```bash
/data/local/tmp/rustfrida --pid <pid> --verbose
```

## Validation

Verified device-side regression script:

```bash
adb push scripts/spawn_test.js /data/local/tmp/spawn_test.js
adb shell su -c "/data/local/tmp/rustfrida --spawn com.asmzhang.testapp -l /data/local/tmp/spawn_test.js"
```

Expected validation points:

- `Java.ready` fires before app code hook installation
- `MainActivity.stringFromJNI()` return value is replaced
- UI text is updated in `onCreate()` and `onResume()`

## `--watch-so` / ldmonitor

The current tree has switched the `ldmonitor` path toward the KPM backend instead of the older eBPF uprobes flow.

- `--watch-so` depends on device-side KPM support
- related resources live in `ldmonitor-kpm`

If you want this path enabled, you still need to deploy the matching KPM module to the target device.

## QBDI

If you want to enable QBDI:

```bash
cargo build -p qbdi-helper --release --target aarch64-linux-android
cargo build -p agent --release --target aarch64-linux-android --features qbdi
cargo build -p rust_frida --release --target aarch64-linux-android --features qbdi
```

## Notes

- `.cargo/config.toml` is machine-local and should be generated by `scripts/setup_cargo_config.py`
- `build.py`, `loader/build_helpers.py`, and `scripts/setup_cargo_config.py` are the portable build entrypoints
- the currently verified primary target is `aarch64-linux-android`
